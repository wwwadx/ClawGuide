## 🔎 收集策略

### 主动收集
1. **监控关键词**
   - "OpenClaw use case"
   - "openclaw tutorial"
   - "openclaw automation"
   - "openclaw workflow"

2. **社区参与**
   - 加入 Discord (#showcase 频道)
   - 关注 GitHub Discussions
   - 订阅 Reddit r/OpenClaw (如果存在)

3. **用户访谈**
   - 主动询问用户场景
   - 收集配置前后的对比
   - 记录遇到的坑和解决方案

### 被动收集
1. **文档反馈**
   - 当用户说 "这个帮我省了 X 时间"
   - 当用户问 "能做 Y 吗？"
   - 当用户分享他们的配置

2. **问题分析**
   - FAQ 中的常见问题
   - GitHub Issues 的实际需求
   - Discord 帮助频道

## 📝 用例验证流程

当一个潜在用例出现时：

1. **记录**
   - 用户原始描述
   - 使用的 skills
   - 配置步骤

2. **验证**
   - 能否复现？
   - 是否通用？
   - 效果可衡量？

3. **完善**
   - 补充细节
   - 添加示例
   - 标注难度

4. **发布**
   - 添加到 SCENARIOS_QUICK_REF.md
   - 更新 TUTORIALS.md
   - 写成独立教程（如果复杂）

## 🎯 优先级

### 高优先级（立即收集）
- 开发者效率相关（代码、PR、部署）
- 常见表单自动化（购物、填报）
- 内容创作（写作、PPT、图表）

### 中优先级（持续收集）
- 学习和研究
- 数据处理
- 生活自动化

### 低优先级（有空再做）
- 小众工具
- 实验性功能
- 地域特定（非中文用户）
