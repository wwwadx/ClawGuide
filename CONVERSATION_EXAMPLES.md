# 对话式定制示例

> 看看如何用对话来定制 OpenClaw，而不是手动编辑文件

## 💡 核心理念

**"告诉它你要什么，它会自己完成"**

这就是 OpenClaw 的强大之处 - 你不需要了解配置文件的格式、位置或语法，只需要用自然语言表达需求。

---

## 🗣️ 场景 1：修改助手身份

### ❌ 传统方式（手动编辑）

```bash
# 1. 找到文件位置
vim ~/workspace/IDENTITY.md

# 2. 编辑文件
- **Name:** ClawGuide
+ **Name:** 小助手

# 3. 保存并退出
: wq

# 4. 重启 gateway
openclaw gateway restart
```

**问题：**
- 需要知道文件在哪
- 需要会用编辑器
- 需要了解 markdown 格式
- 容易出错

### ✅ OpenClaw 方式（对话）

```
你：把你的名字改成"小助手"

OpenClaw：好的！我来修改配置...

[正在编辑 IDENTITY.md]
✅ 已将名字从 "ClawGuide" 改为 "小助手"

需要我重启 gateway 让改动生效吗？

你：好的

OpenClaw：[重启 gateway]
✅ 完成！我现在叫"小助手"了。
```

**优势：**
- 只需要说话
- 助手自动找到并编辑正确文件
- 立即生效

## 🎯 实用对话模板

### 修改身份相关

```
- "把你的名字改成 [名字]"
- "你叫 [名字] 吧"
- "用 [emoji] 作为你的标志"
- "你的风格要 [正式/随意/幽默/专业]"
```

### 调整行为相关

```
- "回复要 [更简洁/更详细]"
- "不要用 [表情符号/emoji]"
- "每次回复都要 [先分析再回答]"
- "用 [中文/英文] 回复"
```

### 功能相关

```
- "帮我安装 [skill 名字]"
- "搜索一下有没有 [功能描述] 的 skill"
- "添加 [功能] 功能"
- "去掉 [功能] 功能"
```

### 记录相关

```
- "帮我记录一下：[用例描述]"
- "我刚发现一个好用的场景..."
- "把这个添加到用例库"
```

## 🎓 总结

### 关键原则

1. **用对话代替命令**
   - 不要说"编辑这个文件的第 X 行"
   - 而是说"我希望你..."

2. **信任 AI 的能力**
   - 它知道文件在哪里
   - 它知道如何编辑
   - 它知道如何让改动生效

3. **迭代式改进**
   - 先说一个大方向
   - 看效果后微调
   - 逐步优化

4. **保持对话**
   - 定期检查配置
   - 根据使用反馈调整
   - 持续优化

## 🤔 常见问题

**Q: 如果 AI 编辑错了怎么办？**

A:
```
你：撤销刚才的修改

OpenClaw：[使用 git 恢复文件]
✅ 已恢复到修改前的状态。
```

**Q: 我还是想手动编辑怎么办？**

A:
```
你：我想手动编辑 IDENTITY.md

OpenClaw：没问题！文件在 ~/workspace/IDENTITY.md

编辑完成后告诉我，我会重启 gateway。
```

**Q: 对话式定制会失去控制吗？**

A: 不会。OpenClaw 会：
- 告诉你它要做什么
- 等你确认后再执行
- 显示具体的修改内容
- 支持 git 撤销

---

**记住：OpenClaw 是 AI 助手，不是配置文件编辑器。**

**用对话，不要用编辑器。** 🗣️✨
